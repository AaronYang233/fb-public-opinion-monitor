# 聚类算法

## 相似度计算方法

### 1. 关键词匹配法（默认）

**原理：** 提取帖子中的关键词，计算关键词重叠度

**步骤：**
1. 对每条帖子进行分词（中文/英文）
2. 去除停用词（的、是、了、the、is 等）
3. 提取名词、动词作为关键词
4. 计算两条帖子的关键词交集比例

**相似度公式：**
```
similarity = |keywords_A ∩ keywords_B| / |keywords_A ∪ keywords_B|
```

**聚类阈值：**
- similarity ≥ 0.6 → 同一聚类
- similarity ≥ 0.4 → 可能相关，需人工确认
- similarity < 0.4 → 不相关

### 2. 问题分类聚类

**简化聚类法：**

当多条帖子满足以下条件时，视为同一聚类：

| 条件 | 说明 |
|------|------|
| 同一问题分类 | 质量问题 / 功能异常 / 售后问题 / 安全风险 |
| 时间窗口内 | 7 天内发布 |
| 关键词重叠 | 至少 2 个核心关键词相同 |

**示例：**
```
帖子 A: "产品质量太差了，用两天就坏了"
帖子 B: "质量真的不行，做工粗糙"
帖子 C: "功能正常，就是质量有点差"

→ 都属于"质量问题"分类，且都包含"质量"关键词
→ 聚类为同一组
```

### 3. 语义聚类（可选升级）

**使用 embedding 模型：**

```python
# 伪代码
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')

# 计算帖子 embedding
embeddings = model.encode([post1, post2, post3])

# 计算余弦相似度
similarity = cosine_similarity(embeddings)

# 使用 DBSCAN 聚类
clusters = DBSCAN(eps=0.3, min_samples=2).fit(embeddings)
```

**优点：** 能捕捉语义相似（如"电池不行"和"续航短"）
**缺点：** 需要 API 调用，成本较高

## 聚类 ID 生成规则

```
聚类ID = {日期}_{分类缩写}_{序号}

日期: YYYYMMDD
分类缩写: 
  - QL = 质量问题
  - GN = 功能异常
  - SH = 售后问题
  - AQ = 安全风险
  - OT = 其他
序号: 01, 02, 03...

示例: 20240324_QL_01
```

## 聚类分析输出

**聚类报告字段：**

| 字段 | 说明 |
|------|------|
| 聚类ID | 唯一标识 |
| 帖子数量 | 该聚类包含的帖子数 |
| 时间范围 | 最早/最晚帖子时间 |
| 核心关键词 | 出现频率最高的 3-5 个词 |
| 主要问题 | 该聚类的典型问题描述 |
| 严重程度 | 该聚类的最高严重程度 |
| 建议措施 | 针对该聚类的处理建议 |

## 聚类触发预警条件

| 条件 | 预警级别 |
|------|---------|
| 聚类帖子数 ≥ 5 | 高 |
| 聚类帖子数 ≥ 3 且包含严重级别 | 高 |
| 聚类帖子数 ≥ 3 | 中 |
| 聚类帖子数 = 2 且均为强负面 | 低 |
